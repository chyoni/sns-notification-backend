# ADR-004: Testcontainers 기반 통합 테스트 전략

| 항목 | 내용 |
|------|------|
| 상태 | 승인됨 (Accepted) |
| 날짜 | 2026-04-09 |
| 의사결정자 | cwchoiit |

## 컨텍스트 (Context)

ADR-003에서 MongoDB와 MySQL 두 가지 스토리지를 모두 사용하기로 결정했다. 어댑터 계층에 대한 통합 테스트를 작성할 때, 실제 데이터베이스에 가능한 가까운 환경에서 테스트하는 것이 최우선 원칙이다.

테스트 환경 선택지:
1. H2 In-Memory Database
2. Embedded MongoDB
3. Testcontainers (Docker 기반 실제 컨테이너)

## 결정 (Decision)

통합 테스트에 **Testcontainers**를 사용하여 실제 DB와 동일한 엔진으로 테스트한다.

```java
@Testcontainers
class NotificationMongoRepositoryAdapterTest {

    @Container
    static MongoDBContainer mongoDb = new MongoDBContainer("mongo:7.0");

    static {
        System.setProperty("notification.mongo.uri",
            mongoDb.getReplicaSetUrl("notification"));
    }
}
```

`docker-compose.yml`에서 사용하는 MongoDB 버전(`mongo:7.0`)과 동일한 버전을 테스트 컨테이너에도 사용한다. 이를 통해 프로덕션 환경과 테스트 환경의 DB 동작이 일치함을 보장한다.

**의존성 구성** (`notification-core/build.gradle`):
```groovy
testImplementation(platform(libs.testcontainers.bom))
testImplementation(libs.testcontainers.mongodb)
testImplementation(libs.testcontainers.junit)
testImplementation(libs.spring.boot.testcontainers)
```

**테스트 구조:**
- `NotificationMongoRepositoryAdapterTest` — Testcontainers를 사용한 통합 테스트 (실제 MongoDB 7.0)
- `NotificationInMemoryRepositoryAdapterTest` — 순수 단위 테스트 (Spring 컨텍스트 없음)

## 대안 (Alternatives Considered)

**H2 In-Memory Database:**
- MongoDB 테스트가 불가능하다. MySQL과의 SQL 방언 차이로 완전한 호환을 보장할 수 없다.
- 두 가지 서로 다른 스토리지를 사용하는 이 프로젝트에 적합하지 않다.
- 채택하지 않음.

**Embedded MongoDB (flapdoodle):**
- Spring Boot 3.x에서 공식 지원이 중단되었다. MongoDB 버전 지원이 제한적이어서 프로덕션 환경과 동일한 버전 보장이 어렵다.
- 채택하지 않음.

**외부 DB 서버에 직접 연결 (로컬 설치 DB):**
- 개발자마다 환경이 달라 일관성 없는 테스트 결과가 나올 수 있다. CI 환경 구성이 복잡하다.
- 채택하지 않음.

## 결과 (Consequences)

### 긍정적 결과
- 프로덕션 환경과 동일한 DB 엔진(MongoDB 7.0)으로 테스트하므로 신뢰도가 높다.
- MongoDB와 MySQL(예정) 모두 동일한 방식으로 통합 테스트를 작성할 수 있다.
- 테스트 환경 구성을 코드로 관리하므로, 팀원 누구나 별도 환경 설정 없이 테스트를 실행할 수 있다.

### 부정적 결과 / 트레이드오프
- Docker가 반드시 필요하다. Docker 없는 환경(일부 CI 환경)에서는 추가 설정이 필요하다.
- 컨테이너 기동 시간으로 인해 H2보다 테스트 실행 속도가 느리다.
- `System.setProperty`를 정적 블록에서 호출하는 방식은 병렬 테스트 실행 시 상태 간섭 위험이 있다.

### 후속 과제
- `System.setProperty` 정적 블록을 `@DynamicPropertySource`로 교체하여 Spring의 권장 방식 사용
- MySQL JPA 어댑터 구현 시 Testcontainers MySQL 모듈 추가
- CI 환경(GitHub Actions 등)에서 Docker-in-Docker 없이 동작하는지 확인
